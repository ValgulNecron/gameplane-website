<div class="grid gap-4 md:grid-cols-2 md:gap-8">
  <div class="flex flex-col gap-4">
    <div class="flex flex-col gap-1">
      <h3 class="!m-0">Do I need to know Kubernetes?</h3>
      <p>
        One-time cluster setup benefits from basic Helm familiarity. Day-to-day server management —
        console, files, backups, players — is entirely dashboard-driven; kubectl is optional.
      </p>
    </div>
    <div class="flex flex-col gap-1">
      <h3 class="!m-0">Is Gameplane production-ready?</h3>
      <p>
        It's in beta (v0.2.0-beta.4). Core workflows are feature-complete and tested, but not yet
        recommended for unattended production — see the <a href="./roadmap/">Roadmap</a>.
      </p>
    </div>
    <div class="flex flex-col gap-1">
      <h3 class="!m-0">Is it free?</h3>
      <p>Yes. Gameplane is open-source under AGPL-3.0-or-later, with no paid tiers.</p>
    </div>
  </div>
  <div class="flex flex-col gap-4">
    <div class="flex flex-col gap-1">
      <h3 class="!m-0">What happens to my data if I uninstall?</h3>
      <p>
        Depends on your cluster's StorageClass reclaim policy — Gameplane doesn't force-delete PVCs
        itself. Back up critical data before uninstalling.
      </p>
    </div>
    <div class="flex flex-col gap-1">
      <h3 class="!m-0">What database does it use?</h3>
      <p>SQLite by default, or PostgreSQL — selectable at install time via a Helm value.</p>
    </div>
    <div class="flex flex-col gap-1">
      <h3 class="!m-0">Can I manage servers without the dashboard?</h3>
      <p>
        Yes. Advanced users can bypass the API entirely and manage everything with kubectl apply —
        the operator is authoritative, so both stay in sync.
      </p>
    </div>
  </div>
</div>
