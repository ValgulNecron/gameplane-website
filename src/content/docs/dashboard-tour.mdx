## Orientation

The dashboard is a cluster-scoped control surface. Use the top bar to change context without
losing your place.

- Select the cluster and namespace before inspecting or changing resources.
- Use global search to jump between servers, modules, users, and operational surfaces.
- Treat status chips as the current reconciled state, not an optimistic request result.

## Read the fleet at a glance

The home view combines live infrastructure health with product activity so you can spot drift
before opening a resource.

- Check server and backup phases alongside cluster capacity.
- Review recent activity before making a second change to the same resource.
- Follow events and logs when a server is still reconciling or has failed to start.

## Your first operating loop

A healthy workflow moves from intent to reconciliation, then closes the loop with observability
and recovery.
